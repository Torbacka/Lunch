```python
def _job_name(team_id, channel_id):
    return "poll_{}_{}".format(team_id, channel_id)
```

3. **_run_poll(team_id, channel_id)**: signature change — channel is now required, no kwarg default. Body:
```python
def _run_poll(team_id, channel_id):
    with _app.app_context():
        from lunchbot.services import poll_service
        poll_service.push_poll(channel_id, team_id, trigger_source='scheduled')
```
Delete any `poll_channel_for(team_id)` fallback logic.

4. **update_schedule_job(team_id, channel_id, time_val, timezone, weekdays)**:
   - Build `CronTrigger` from `time_val` (datetime.time), `timezone` (string), `weekdays` (comma-separated like existing format).
   - Remove any existing job with name `_job_name(team_id, channel_id)` if present.
   - Add a new job with `args=(team_id, channel_id)`, `id=_job_name(...)`, `replace_existing=True`.

5. **remove_schedule_job(team_id, channel_id)** (NEW):
```python
def remove_schedule_job(team_id, channel_id):
    job_id = _job_name(team_id, channel_id)
    try:
        _scheduler.remove_job(job_id)
    except Exception:
        pass
```

6. **Delete** any function named `poll_channel_for` if it lives in scheduler_service (it may live in workspace_client — leave it there if so; just remove scheduler_service's call to it).

The assumption is verified by the existing `BackgroundScheduler()` call in init_scheduler (see acceptance criterion).

Use snake_case, no type hints, minimal error handling (except for remove_schedule_job's try/pass). Match existing logging calls (structlog).
