# Phase 07.2 Plan 04: Channel-Keyed Scheduler Rewrite Summary

Channel-keyed APScheduler integration: jobs named poll_{team_id}_{channel_id}, loaded from channel_schedules table, with direct channel_id pass-through to push_poll (no poll_channel_for fallback).

## What Changed

### Task 1: Rewrite scheduler_service for channel-keyed jobs (e1cbfd6)

Rewrote `lunchbot/services/scheduler_service.py` to implement D-13:

- **_job_name(team_id, channel_id)**: New helper returning `poll_{team_id}_{channel_id}` format, used in all 4 job-id construction sites.
- **load_all_schedules()**: Now queries `workspaces` for active team_ids, then calls `db_client.list_channel_schedules(team_id)` per team. Creates one job per (team_id, channel_id) row. Removed all reads of `poll_schedule_*` columns and `poll_channel`.
- **_ensure_job(team_id, channel_id, schedule_time, timezone, weekdays)**: Replaces old `_add_job`. Handles both comma-separated string and list weekday formats.
- **update_schedule_job(team_id, channel_id, time_val, timezone, weekdays)**: Signature changed -- `channel_id` is now a required positional arg (was optional kwarg `channel=None`).
- **remove_schedule_job(team_id, channel_id)**: Signature changed to accept channel_id. Uses try/except for idempotent removal.
- **_run_poll(team_id, channel_id)**: Signature changed -- `channel_id` is required, no `poll_channel_for` fallback. Imports only `push_poll` (removed `poll_channel_for` import).
- **init_scheduler(app)**: Unchanged externally -- still uses `BackgroundScheduler()` (in-memory jobstore, confirming no orphan sweep needed).

## Deviations from Plan

None -- plan executed exactly as written.
