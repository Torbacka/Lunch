Slash command payload (Slack form POST to /slack/command):
```
team_id=T123ABC
command=/lunch
text=help   (or empty string for poll trigger)
channel_id=C123
user_id=U123
response_url=https://...
```

Interactive action payload (JSON in form field 'payload'):
```json
{
  "type": "block_actions",
  "team": {"id": "T123ABC"},
  "user": {"id": "U456"},
  "channel": {"id": "C123"},
  "message": {"ts": "1234567890.123456", "blocks": [...]},
  "actions": [{"type": "button", "value": "42"}]
}
```
NOTE: action value is the poll_option_id (integer as string), NOT place_id.
The old mongo code used place_id; the new PostgreSQL code uses poll_option_id.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Slack → /slack/command | Slash command form POST; team_id and channel_id come from Slack |
| Slack → /action | Interactive action payload; user_id and poll_option_id come from button value |
| app → Slack API | update_message sends rebuilt blocks back to Slack |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Spoofing | POST /slack/command | mitigate | Slack signature verification middleware (already wired in __init__.py) rejects unsigned requests |
| T-03-07 | Tampering | poll_option_id in button value | mitigate | int() cast on poll_option_id; malformed values raise ValueError caught at blueprint level returning 400; toggle_vote uses parameterized SQL |
| T-03-08 | Elevation of Privilege | team_id in slash command form | accept | team_id is already extracted by tenant middleware from the same form data; RLS enforces workspace isolation at DB level |
| T-03-09 | Information Disclosure | voter profile cache | accept | profile_cache is in-process memory; data is non-sensitive display names and avatar URLs |
| T-03-10 | Denial of Service | synchronous push_poll | accept | Phase 3 synchronous is acceptable; async job queue is a Phase 5 concern; Slack 3-second timeout drives ephemeral fallback |
| T-03-11 | Tampering | Block Kit blocks rebuilt from DB | mitigate | Blocks are rebuilt from DB data (not from Slack payload blocks), so a tampered payload cannot inject malicious content into the message |
</threat_model>
