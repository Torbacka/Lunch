External select options response shape (Slack requirement):
```json
{"options": [
  {"text": {"type": "plain_text", "text": "Restaurant Name open 4.5"}, "value": "ChIJplace_id"}
]}
```
NOTE: The value is place_id (for the suggestion handler), not the DB restaurant id.
When the user selects an option, the action payload type is 'external_select' and
action.selected_option.value = place_id. The /action handler then calls suggest(place_id).

suggest(place_id) in the new stack:
- Fetch restaurant from DB by place_id
- If url missing, call places_client.get_details(place_id) and call db_client.update_restaurant_url()
- Call db_client.upsert_suggestion(date.today(), restaurant_id)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Google Places API | HTTPS request; API key in query param |
| Slack → /find_suggestions | Search string from user input; passes through to Places API |
| app → PostgreSQL | save_restaurants and add_emoji write user-sourced data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Information Disclosure | GOOGLE_PLACES_API_KEY | mitigate | Key read from config/env var only; never logged; requests library sends as HTTPS query param |
| T-03-13 | Tampering | search string from Slack payload | mitigate | search string passed as Places API keyword parameter (not SQL); SQL uses parameterized queries in save_restaurants; no injection surface |
| T-03-14 | Information Disclosure | Places API response | accept | Restaurant metadata (name, rating, hours) is public data; no PII in Places response |
| T-03-15 | Spoofing | POST /find_suggestions | mitigate | Slack signature middleware (already wired) validates request origin; unsigned requests rejected |
| T-03-16 | Denial of Service | search_and_update_emoji | accept | /emoji is an internal endpoint (no Slack signature path); Phase 3 leaves it as a manual trigger; automated scheduling with rate-limiting is Phase 5 |
| T-03-17 | Tampering | place_id from Slack action payload | mitigate | place_id used in parameterized SQL (db_client.get_restaurant_by_place_id); no string interpolation into SQL |
</threat_model>
