    - get_bot_token(team_id) returns decrypted bot token; raises ValueError if workspace not found or inactive
    - post_message(channel, blocks, team_id, text='') calls https://slack.com/api/chat.postMessage with Authorization: Bearer {token}; returns response JSON
    - update_message(channel, ts, blocks, team_id) calls https://slack.com/api/chat.update; returns response JSON
    - get_user_profile(user_id, team_id) calls https://slack.com/api/users.profile.get; returns profile dict
    - All functions use a requests.Session() at module level for connection reuse
    - Logging: log "Status code: {status} response: {ok}" for each call at INFO level (matches existing convention)

  <action>
Create `lunchbot/client/slack_client.py`. This is the multi-tenant replacement for
`service/client/slack_client.py`. The key difference: every function takes `team_id` and
resolves the workspace bot token via `get_workspace(team_id)` + `decrypt_token()`.

Implementation steps:
1. Import: requests, logging, current_app, get_workspace from workspace_client, decrypt_token from oauth.
2. Module-level `session = requests.Session()` and `SLACK_API = "https://slack.com/api/"`.
3. `get_bot_token(team_id)`: call get_workspace(team_id); if None or not is_active, raise ValueError(f"No active workspace: {team_id}"); decrypt and return bot_token_encrypted using current_app.config['FERNET_KEY'].
4. `_headers(token)`: return {'Content-Type': 'application/json;charset=utf-8', 'Authorization': f'Bearer {token}'}.
5. `post_message(channel, blocks, team_id, text='')`: call get_bot_token; POST to chat.postMessage with json={'channel': channel, 'blocks': blocks, 'text': text}; log status+ok; return response.json().
6. `update_message(channel, ts, blocks, team_id)`: call get_bot_token; POST to chat.update with json={'channel': channel, 'ts': ts, 'blocks': blocks, 'as_user': True}; log status+ok; return response.json().
7. `get_user_profile(user_id, team_id)`: call get_bot_token; POST to users.profile.get with data={'user': user_id}, Content-Type: application/x-www-form-urlencoded; return response.json()['profile'].

Write tests in `tests/test_slack_client.py` using unittest.mock.patch on `requests.Session.post`
and `lunchbot.client.slack_client.get_workspace`. Test: token resolution, post_message call shape,
update_message call shape, get_user_profile return, ValueError on missing workspace.
Use app fixture from conftest.py for app context (FERNET_KEY available).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Slack API | Bot token travels in Authorization header over HTTPS |
| app → PostgreSQL | Encrypted token read from DB; decryption happens in app memory |
| config → app memory | FERNET_KEY loaded from env var; must not be logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | get_bot_token() | mitigate | Never log the decrypted token; log only team_id at DEBUG level |
| T-03-02 | Information Disclosure | _headers() | mitigate | Do not log the full headers dict; the Authorization value is a secret |
| T-03-03 | Spoofing | Slack API responses | accept | Responses come over HTTPS from slack.com; no request forgery surface here |
| T-03-04 | Denial of Service | requests.Session reuse | accept | Session reuse is safe; Slack API rate-limits per token, not per connection |
| T-03-05 | Tampering | FERNET_KEY in config | mitigate | FERNET_KEY is only read from env var; never hardcoded; TestConfig uses generated key |
</threat_model>
