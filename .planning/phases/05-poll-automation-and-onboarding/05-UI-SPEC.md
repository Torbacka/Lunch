# Phase 5 — UI Design Contract

> Visual and interaction contract for Slack Block Kit surfaces. Generated by gsd-ui-researcher.

**Platform note:** LunchBot is a Slack bot. All UI surfaces are Slack Block Kit (App Home tab, modals, channel messages). No HTML/CSS/web components involved. This contract specifies Block Kit layout, element types, copy, and interaction patterns instead of traditional web design tokens.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Slack Block Kit) |
| Preset | not applicable |
| Component library | Slack Block Kit |
| Icon library | Slack standard emoji |
| Font | Slack platform default (not configurable) |

**Note:** Spacing, typography, and color are controlled by Slack's rendering engine. This contract specifies Block Kit block types, element choices, and layout structure instead of pixel values.

---

## Spacing Scale

Slack Block Kit controls spacing. The contract specifies structural spacing via block types:

| Structural Element | Block Kit Equivalent | Usage |
|--------------------|---------------------|-------|
| Section gap | `divider` block | Separate setting groups on App Home |
| Inline gap | `context` block | Secondary info below a setting value |
| Group padding | `header` block | Label each settings group |
| Page-level break | `divider` block + `header` block combo | Major sections on App Home |

Exceptions: none — Slack enforces its own spacing.

---

## Typography

Slack Block Kit controls all font rendering. The contract specifies text formatting:

| Role | Block Kit Type | Formatting |
|------|---------------|------------|
| Page title | `header` block | Plain text, auto-bold by Slack |
| Section heading | `section` block with `mrkdwn` | `*bold*` text |
| Body / value display | `section` block with `mrkdwn` | Plain mrkdwn |
| Secondary / metadata | `context` block | Smaller, muted by Slack |
| Button label | `button` element | `plain_text`, max 24 characters |
| Modal title | `title` in modal view | `plain_text`, max 24 characters |

---

## Color

Slack Block Kit controls all colors. The contract specifies semantic usage via element types:

| Role | Block Kit Mechanism | Usage |
|------|-------------------|-------|
| Dominant surface | App Home tab background | Slack renders this automatically |
| Secondary surface | `section` blocks | Settings values, grouped content |
| Accent (primary action) | `button` with `style: "primary"` | "Begin Setup" CTA on first-time install only |
| Destructive | `button` with `style: "danger"` | "Remove Schedule" action only |
| Neutral action | `button` with no style | All edit buttons ("Edit Channel", "Edit Schedule", etc.) |

**Primary-styled button reserved for:** "Begin Setup" button on the first-time onboarding state only. All other buttons use default (neutral) style to avoid visual noise.

**Danger-styled button reserved for:** "Remove Schedule" confirmation inside the schedule modal only.

## Block Kit Surfaces

### Surface 1: App Home Tab — Settings Panel (BOT-10)

The App Home tab renders as a single settings panel. Two states: unconfigured (first install) and configured (ongoing use).

#### State A: First-Time Install (no settings configured)

```
[header]      LunchBot Settings
[divider]
[section]     :wave: *Welcome to LunchBot!*
              Set up your workspace to get started with lunch polls.
[actions]     [Begin Setup] (primary style)
[divider]
[context]     Only workspace admins can change these settings.
```

**"Begin Setup" button:** Opens the Channel Selection modal (Surface 3a). After the channel is saved, the App Home refreshes to State B with remaining settings showing defaults or empty.

#### State B: Configured Workspace

```
[header]      LunchBot Settings
[divider]

[section]     :hash: *Poll Channel*
              #lunch-votes                          [Edit] (button)
[divider]

[section]     :clock3: *Poll Schedule*
              (one of two sub-states below)
[context]     Polls post automatically at the scheduled time.

  Sub-state B1 — No schedule configured:
[section]     No schedule configured. Polls are triggered manually    [Set Schedule] (button)
              with `/lunch`.

  Sub-state B2 — Schedule configured:
[section]     11:30 AM, Europe/Stockholm              [Edit] (button)
              Mon, Tue, Wed, Thu, Fri
[actions]                                             [Remove Schedule] (danger style, no confirm — opens modal)

[divider]

[section]     :bar_chart: *Poll Size*
              5 options (2 smart picks, 3 random)    [Edit] (button)
[divider]

[section]     :round_pushpin: *Location*
              Stockholm, Sweden                       [Edit] (button)
[divider]

[context]     Only workspace admins can change these settings.
```

**Layout rules:**
- Each setting group: one `section` block (label + value + Edit button), optionally followed by a `context` block for secondary info
- Groups separated by `divider` blocks
- Edit buttons use `overflow` or `button` accessory on the `section` block (button preferred for clarity)
- Schedule section includes a `context` block explaining what the schedule does

#### Admin gating

All edit buttons are visible to all users, but the `app_home_opened` handler checks if the user is a Slack workspace admin. Non-admins see a read-only version: same layout, but buttons are replaced with `context` blocks saying "Contact a workspace admin to change settings."

### Surface 2: Poll Message (existing — no changes)

The poll message posted to channels is unchanged from Phase 3. The scheduler (BOT-09) calls `push_poll()` which uses `build_poll_blocks()` as-is. No visual changes needed.

### Surface 3: Configuration Modals (BOT-09, BOT-10)

All settings are edited via modals opened from App Home edit buttons. Each modal uses `views.open` with `trigger_id` from the button click.

#### Surface 3a: Channel Selection Modal

```
Modal title:    Poll Channel
Submit button:  Save
Close button:   Cancel

[input]         Channel
                type: conversations_select
                filter: { include: ["public", "private"], exclude_bot_users: true }
                initial_conversation: (current value if set)
```

**Validation:** Channel must be selected. On submit, bot verifies it can post to the channel (is a member). If not, returns a modal error: "LunchBot needs to be invited to this channel first. Type `/invite @LunchBot` in the channel."

#### Surface 3b: Schedule Modal

```
Modal title:    Poll Schedule
Submit button:  Save Schedule
Close button:   Cancel

[input]         Time
                type: timepicker
                initial_time: (current value or "11:30")
                placeholder: "Select time"

[input]         Timezone
                type: static_select
                options: [top 30 IANA timezones, grouped by region]
                initial_option: (current value or "Europe/Stockholm")
                placeholder: "Select timezone"

[input]         Days
                type: checkboxes
                options: [Mon, Tue, Wed, Thu, Fri, Sat, Sun]
                initial_options: (current values or [Mon, Tue, Wed, Thu, Fri])
```

**Validation:** At least one day must be selected. Time is required. Timezone is required.

**Timezone list (30 options, grouped):**
- Americas: America/New_York, America/Chicago, America/Denver, America/Los_Angeles, America/Toronto, America/Sao_Paulo, America/Mexico_City
- Europe: Europe/London, Europe/Paris, Europe/Berlin, Europe/Stockholm, Europe/Amsterdam, Europe/Madrid, Europe/Rome, Europe/Warsaw, Europe/Moscow
- Asia: Asia/Tokyo, Asia/Shanghai, Asia/Kolkata, Asia/Singapore, Asia/Dubai, Asia/Seoul, Asia/Bangkok, Asia/Jakarta
- Pacific: Australia/Sydney, Australia/Melbourne, Pacific/Auckland, Pacific/Honolulu
- Africa: Africa/Cairo, Africa/Lagos

#### Surface 3c: Poll Size Modal

```
Modal title:    Poll Options
Submit button:  Save
Close button:   Cancel

[input]         Total poll options
                type: static_select
                options: [3, 4, 5, 6, 7, 8]
                initial_option: (current value or "5")

[input]         Smart picks (Thompson sampling)
                type: static_select
                options: [0, 1, 2, 3, 4]
                initial_option: (current value or "2")

[context]       Smart picks use your team's voting history to suggest
                restaurants you're likely to enjoy. The rest are random.
```

**Validation:** Smart picks must be less than total poll options. If smart picks >= total, return modal error: "Smart picks must be fewer than total options, so there's room for new discoveries."

#### Surface 3d: Location Modal

```
Modal title:    Search Location
Submit button:  Save
Close button:   Cancel

[input]         Location
                type: plain_text_input
                initial_value: (current value or "")
                placeholder: "e.g. Stockholm, Sweden"

[context]       This location is used to search for nearby restaurants
                via Google Places.
```

**Validation:** Location must not be empty.

#### Surface 3e: Remove Schedule Confirmation Modal

Triggered from "Remove Schedule" danger button on App Home.

```
Modal title:    Remove Schedule
Submit button:  Remove (danger style text)
Close button:   Keep Schedule

[section]       Are you sure you want to remove the automatic poll schedule?
                Polls will only be triggered manually with `/lunch`.
```

**On confirm:** Clears `poll_schedule_time`, `poll_schedule_timezone`, `poll_schedule_weekdays` to NULL. Removes the APScheduler job. Refreshes App Home to show "No schedule configured" sub-state.

## Interaction Contract

| Trigger | Action | Result |
|---------|--------|--------|
| User opens App Home tab | `app_home_opened` event fires | Render settings panel (State A or B based on config) |
| User clicks "Begin Setup" | `block_actions` with button action | Open Channel Selection modal (Surface 3a) |
| User clicks any "Edit" button | `block_actions` with button action | Open corresponding modal (3a-3d) |
| User clicks "Remove Schedule" | `block_actions` with button action | Open Remove Schedule confirmation modal (3e) |
| User submits any modal | `view_submission` event | Validate, save to DB, update APScheduler if schedule changed, refresh App Home via `views.publish` |
| Modal validation fails | `view_submission` handler returns `response_action: errors` | Inline error shown on the failing input field |
| Scheduled poll triggers | APScheduler fires cron job | Calls `push_poll(channel, team_id)` — same as manual `/lunch` |
