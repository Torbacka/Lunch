## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Paragraph text, privacy policy content, support page text |
| Label | 14px | 400 | 1.4 | Section subheadings in privacy policy, footer links, step titles in "How it works" |
| Heading | 20px | 600 | 1.2 | Page headings (h1) for privacy and support, section headings on landing |
| Display | 32px | 600 | 1.2 | Landing page hero heading only |

2 weights declared: 400 (regular) and 600 (semibold). Label at 14px is visually distinct from Body at 16px through size alone — no intermediate weight needed.

Source: Body 16px/400, Heading 20px/600 match existing `oauth.py` and `setup.py`. Display 32px is new for the landing page hero — justified because the landing page is a marketing surface, not a utility page.

## Layout Contract

Primary focal point: hero display heading + "Add to Slack" button — largest text on page (32px/600), positioned directly below nav, drawing the eye first on page load.

### Landing Page (`/`)

| Property | Value |
|----------|-------|
| Container max-width | 720px |
| Container horizontal padding | 16px |
| Container centering | margin: 0 auto |

Sections (top to bottom):

1. **Nav bar** — "LunchBot" text left, links (Privacy, Support) right. Full-width, 16px vertical padding, border-bottom #D1D5DB.
2. **Hero** — Display heading (32px/600), body paragraph (16px/400), "Add to Slack" button. 64px top padding, 48px bottom padding.
3. **How it works** — 3-step numbered list. Each step: Label (14px/400) for step title, Body (16px/400) for description. 48px top margin from hero.
4. **Footer** — Links to Privacy and Support. Muted text (#6B7280) at 14px. 48px top margin, 24px bottom padding. Border-top #D1D5DB.

### Privacy Policy (`/privacy`)

| Property | Value |
|----------|-------|
| Container max-width | 640px |
| Container horizontal padding | 16px |

Sections: Nav bar (same as landing) then prose content. Use Heading (20px/600) for page title, Label (14px/400) for section subheadings, Body (16px/400) for content. Unordered lists for data points. 16px margin between paragraphs. 24px margin before each section subheading.

### Support Page (`/support`)

| Property | Value |
|----------|-------|
| Container max-width | 480px |
| Container horizontal padding | 16px |

Sections: Nav bar (same as landing) then simple content. Heading (20px/600) for page title, Body (16px/400) for description and email link. Minimal page — no sections needed beyond heading + body + email link.

### Shared Nav Component

Present on all three pages. Inline HTML, not a template partial (per D-01).

```
LunchBot [text, 16px, 600, #111827, links to /]    Privacy [14px, 400, #4A154B, links to /privacy]  Support [14px, 400, #4A154B, links to /support]
```

Full-width bar, max-width content centered. 16px vertical padding. Bottom border 1px solid #D1D5DB.

### Privacy Policy Content Sections

Per D-08, D-09, D-10, the privacy policy must include these sections in this order:

1. **What LunchBot Does** — One-sentence description of the service.
2. **Data We Collect** — Bulleted list:
   - Workspace ID and team name (stored in database)
   - User display names and avatar URLs (cached in memory during active sessions, not persisted to database)
   - Vote history including user ID and restaurant selections (stored in database)
   - Encrypted bot token (Fernet-encrypted, stored in database)
   - Restaurant data from Google Places API: name, address, rating, place ID (cached in database)
3. **How We Use Your Data** — To operate the lunch poll service. No advertising. No selling data to third parties.
4. **Data Retention** — Data retained while LunchBot is installed in your workspace. After uninstall, workspace is soft-deleted (tokens cleared, workspace marked inactive). No automated purge of historical vote data.
5. **Data Deletion** — Uninstalling LunchBot from your Slack workspace triggers automatic soft-deletion. For full data removal requests, contact {support_email}.
6. **Third-Party Services** — Google Places API (restaurant data), Slack API (messaging and authentication). Links to their respective privacy policies.
7. **Contact** — Questions about this policy: {support_email}.

## "Add to Slack" Button Specification

Per D-06, use the official Slack button assets:

```html
<a href="/slack/install">
  <img alt="Add to Slack" height="40" width="139"
       src="https://platform.slack-edge.com/img/add_to_slack.png"
       srcset="https://platform.slack-edge.com/img/add_to_slack.png 1x, https://platform.slack-edge.com/img/add_to_slack@2x.png 2x">
</a>
```

Do NOT create a custom-styled button for this CTA. The official image is required for Slack marketplace compliance.

## Responsive Behavior

All pages use a single breakpoint strategy: content is max-width constrained and horizontally padded. No media queries needed — the inline layout is mobile-friendly by default.

| Viewport | Behavior |
|----------|----------|
| < 720px | Content fills width minus 16px horizontal padding |
| >= 720px | Content centered at max-width, padding preserved |

The nav bar text links stack acceptably at narrow widths due to natural line wrapping. No hamburger menu — only 2 nav links.
