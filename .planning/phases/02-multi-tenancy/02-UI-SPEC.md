# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Multi-Tenancy. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Surface Assessment

**Phase 2 is a backend-only phase.** The requirements (MTNT-01 through MTNT-04) cover Slack OAuth V2, PostgreSQL Row-Level Security, tenant middleware, and uninstall handling. There is no custom frontend application, no React/Next.js/Vite stack, and no browser-rendered UI beyond two minimal OAuth endpoints.

**UI touchpoints in this phase (2 total):**

1. **OAuth install redirect** (`/slack/install`) — HTTP 302 redirect to Slack's authorize URL. No rendered page.
2. **OAuth success page** (`/slack/oauth_redirect`) — Minimal HTML response after successful installation. Single page, no navigation.

All user-facing interactions (polls, voting, slash commands) happen inside Slack via Block Kit, which is Slack's design system — not ours to specify.

## OAuth Success Page Specification

Since this is the only rendered HTML in Phase 2, here is the complete specification.

**Route:** `GET /slack/oauth_redirect` (after successful token exchange)

**Layout:**
- Centered content, max-width 480px
- System font stack: `system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- Body padding: 16px
- Content vertical margin: 24px from viewport top

**Content (top to bottom):**
1. Heading: "LunchBot Installed" — 20px, weight 600, color #111827
2. Body text: "LunchBot has been added to your workspace. You can close this tab and return to Slack to start using it." — 16px, weight 400, color #374151, line-height 1.5
3. Link: "Return to Slack" — 16px, weight 400, color #4A154B, underline on hover

**Error variant** (when OAuth exchange fails):
1. Heading: "Installation Failed" — 20px, weight 600, color #DC2626
2. Body text: "Something went wrong connecting to your Slack workspace. Please try again from the Add to Slack button, or contact support if the problem persists." — 16px, weight 400, color #374151, line-height 1.5

**No JavaScript required.** Plain semantic HTML with inline styles or a single `<style>` block.
