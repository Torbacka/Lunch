# Phase 07.1 Plan 05: Always-prompt + Add-office modal Summary

Unbound-channel `/lunch` now always prompts (regardless of office count), includes an "Add a new office" button that opens a Places-backed Slack modal, and submitting the modal creates a `workspace_locations` row + binds the current channel in a single flow.

## What Shipped

### Task 1 â€” Always-prompt prompt + modal builder

**`lunchbot/services/office_modal_service.py` (new)**
- `build_add_office_modal(team_id, channel_id=None)` returns a Slack modal with:
  - `callback_id='modal_add_office'`
  - An `input` block containing an `external_select` (`action_id='office_search_select'`, `min_query_length=3`)
  - `private_metadata = json.dumps({'team_id', 'channel_id'})` â€” `channel_id` omitted for the App Home flow (plan 06)
- Constants exported: `CALLBACK_ADD_OFFICE`, `OFFICE_SEARCH_SELECT`, `OFFICE_SEARCH_BLOCK`

**`lunchbot/blueprints/polls.py`**
- New constant `CHANNEL_LOC_ADD_OFFICE = 'channel_loc_add_office'`
- `_build_channel_location_prompt` rewritten:
  - Always prompts (no single-office shortcut, D-05)
  - Removes the "Use default office" button (D-05)
  - Always appends an "Add a new office" button
  - Zero-office workspaces get only the Add button with a distinct intro text (D-09)
- Dropped the now-unused `get_default_location` import

### Task 2 â€” Slack-side wiring

**`lunchbot/blueprints/slack_actions.py`**
- Imports extended: `create_workspace_location`, `list_workspace_locations`, `CHANNEL_LOC_ADD_OFFICE`, and the modal-service constants/builder.
- `_handle_block_actions`: new branch â€” `channel_loc_add_office` button opens `build_add_office_modal` via `slack_client.views_open`.
- `_handle_view_submission`: new `modal_add_office` branch:
  - Extracts `place_id` from the modal state, rejects empty/unresolvable with `response_action: errors` (T-07.1-20).
  - Calls `places_client.get_place_details`, pulls `geometry.location`, composes an office name from `name` + first line of `formatted_address`.
  - `list_workspace_locations` checks whether this is the first office; if so, the new row is marked `is_default=True` (invariant: exactly one default).
  - `create_workspace_location` â†’ `structlog.info('office_create', team_id, location_id, actor_user_id, via=...)` with `via='lunch_prompt'` when `channel_id` present, else `'app_home'` (D-24).
  - When `channel_id` present: `bind_channel_location` + `poll_service.push_poll(trigger_source='add_office_modal')`.
- `find_suggestions`: new branch BEFORE the restaurant search â€” when `payload['action_id'] == 'office_search_select'`, proxy `places_client.autocomplete` and return up to 20 predictions as Slack `options` (`text=description[:75]`, `value=place_id`). Existing restaurant search path is untouched.

## Threat Mitigations Applied

| Threat | Mitigation |
|---|---|
| T-07.1-19 Spoofing (new action ids) | New action ids and `/action` / `/find_suggestions` routes still flow through `lunchbot/middleware/signature.py` â€” no SKIP_PATHS changes. |
| T-07.1-20 Tampering (place_id) | User-submitted `place_id` is only passed into `places_client.get_place_details`; missing `geometry.location` â†’ `response_action: errors`. Users cannot inject arbitrary lat/lng. |
| T-07.1-21 private_metadata disclosure | `private_metadata` carries `team_id` + `channel_id`, both of which the user already knows. |
| T-07.1-22 Non-admin privilege | D-07/D-11 accepts non-admin office creation; logged with `via=lunch_prompt`. |
| T-07.1-24 Cross-tenant search | Google Places data is global; `create_workspace_location` still scopes via `_set_tenant` (plan 02). |

## Decisions Realized

- **D-05 (always prompt):** `_build_channel_location_prompt` no longer branches on office count for the "should we prompt" question â€” the caller (`slash_command`) already guards with `resolve_location_for_channel`. Once unbound, we always return an ephemeral prompt.
- **D-06 (Add button):** The Add button is rendered unconditionally (both zero- and Nâ‰Ą1-office branches). The button `value` carries the channel id so the `/action` handler can plumb `private_metadata.channel_id` without re-parsing the block payload (defense in depth).
- **D-07/D-11 (non-admin allowed):** No admin check in the view_submission branch.
- **D-09 (zero-office branch):** Distinct intro text + no `static_select` when `list_workspace_locations` returns `[]`.
- **D-10 (Places modal):** External-select with `min_query_length=3` avoids one-character queries; the options endpoint caps at 20 results and truncates `description` to 75 chars (Slack hard limit is 75 for plain_text options).
- **D-23 (first-office default):** First office auto-marked default rather than leaving `set_default_workspace_location` for a follow-up click â€” keeps the "exactly one default" invariant from plan 02 intact.
- **D-24 (audit event):** `office_create` logged via `structlog` with `via=lunch_prompt|app_home`. Plan 06 will reuse the same event from the App Home path.
