# Phase 07.1 Plan 04: Places-autocomplete install form Summary

Rewrote `/slack/setup` so admins pick their office via a Google-Places-backed address picker. On submit the handler resolves the chosen `place_id` to lat/lng + formatted address via `places_client.get_place_details`, creates a `workspace_locations` row with `is_default=True`, and fires background restaurant seeding against the new office. The legacy `workspaces.location` write path is gone and the Google Places API key never reaches the browser.

## What Shipped

**Task 1 — Rewrite `/slack/setup` end-to-end** (`lunchbot/blueprints/setup.py`)

- `setup_submit` now reads `team_id`, `place_id`, and optional `session_token` from the form.
- Missing `team_id`/`place_id` → 400 with form + "Pick an address from the suggestions."
- Unknown `team_id` (no `get_workspace` row) → 400 with form + "Workspace not found. Please reinstall LunchBot." and `setup_unknown_team` structlog warning.
- `places_client.get_place_details(place_id, session_token=...)` resolves the pick; missing `geometry.location.lat/lng` → 400 with "Could not resolve that address. Try a different one." and `setup_place_resolve_failed` structlog warning.
- On success: builds `office_name = f"{name}, {short_address}"` (where `short_address` is the first comma segment of `formatted_address`, collapsed to just `name` when redundant), constructs `lat_lng = f"{lat},{lng}"`, calls `create_workspace_location(team_id, office_name, lat_lng, is_default=True)`, emits `office_create` structlog event with `team_id`, `location_id`, `actor_user_id='install'`, `via='install_flow'`, `place_id`, then starts the daemon `_seed_restaurants` thread and returns `_success_page()`.
- `update_workspace_location` import removed; nothing in `setup.py` touches the legacy `workspaces.location` column anymore.
- `_seed_restaurants` left intact (still takes a `lat,lng` string) — it was already accepting the same shape the new handler builds.

**`_form_page` rewrite** — replaces the raw-coordinate text box with:

- Hidden fields for `team_id`, `place_id`, `session_token`.
- Visible text `<input id="address">` with `autocomplete="off"`.
- Inline JS that debounces to 250ms, calls `/places/autocomplete?q=...&session_token=...`, stores the returned `session_token`, renders `main_text`/`secondary_text` rows in a `.suggestions` panel, and on click writes the chosen `place_id` into the hidden field + enables the Submit button. Outside-clicks collapse the panel.
- Submit button is disabled until a suggestion is picked, preventing free-form address text from ever reaching the server.
- `GOOGLE_PLACES_API_KEY` never appears in the HTML — the browser only talks to `/places/autocomplete` and the server proxies to Google.

**Tests** — `tests/test_setup_install_flow.py` (6 tests, all passing):

1. `test_setup_form_renders_places_widget` — GET renders the Places widget, references `/places/autocomplete`, has `name="place_id"`, and has no legacy `name="coords"` or `placeholder="59.3419"` input.
2. `test_setup_post_creates_office_and_seeds` — happy path: get_place_details called with `session_token='tok'`, `create_workspace_location` called with `(T_X, name-containing-"Spotify HQ", "59.33,18.06", is_default=True)`, background thread started.
3. `test_setup_post_missing_place_id` — 400 when `place_id` absent.
4. `test_setup_post_unknown_workspace` — 400 when `get_workspace` returns None.
5. `test_setup_post_unresolvable_place` — 400 when `get_place_details` returns an empty `result` (no geometry); `create_workspace_location` NOT called.
6. `test_setup_does_not_import_legacy_writer` — static assertion that `lunchbot.blueprints.setup` has no `update_workspace_location` attribute.

## Threat Mitigations Applied

| Threat ID | Mitigation implemented |
|---|---|
| T-07.1-14 (Tampering — POST injection) | `get_workspace(team_id)` gate rejects unknown teams with 400; only a `place_id` is accepted for the office location, never raw lat/lng from the browser. Coordinates come exclusively from `places_client.get_place_details`, so an attacker cannot inject arbitrary points. |
| T-07.1-15 (Info Disclosure — API key) | Form template references only `/places/autocomplete` and `/places/details`. `grep GOOGLE_PLACES_API_KEY lunchbot/blueprints/setup.py` returns 0 matches. |
| T-07.1-17 (Info Disclosure — Places response) | Only `name`, `formatted_address`, `lat`, `lng` are read out of the `get_place_details` payload; the rest of the Google response is discarded before any persistence or logging. |

T-07.1-16 (install-form team_id spoofing) and T-07.1-18 (background thread DoS) remain `accept` per the plan's threat model.
