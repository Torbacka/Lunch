# Phase 07.1 Plan 02: Workspace-client office mutations + resolver contract change Summary

One-liner: Removed single-office auto-bind from `resolve_location_for_channel` and added tenant-scoped rename/delete/set_default helpers with `structlog` audit events, while eliminating every read/write of the legacy `workspaces.location` column from `workspace_client.py`.

## What Was Built

Task 1 rewrites `resolve_location_for_channel` to a pure lookup: if a `channel_locations` row exists it returns the joined row, otherwise `None`. The old "single-office workspaces auto-bind on first /lunch" fallback is gone (D-08). `update_workspace_location` is deleted entirely; `get_workspace_settings` no longer selects `location`; `update_workspace_settings.ALLOWED` no longer accepts `'location'`. The two existing tests that depended on auto-bind have been replaced with tests asserting the new "always-prompt" contract, and `test_slash_command_single_location_still_prompts` no longer patches `get_default_location` in anticipation of plan 05 dropping that import from `polls.py`.

Task 2 adds three helpers, all under `_set_tenant(conn, team_id)` and all emitting `structlog` events (`office_rename`, `office_delete`, `office_set_default`) that capture `team_id`, `location_id`, `actor_user_id`:

- `rename_workspace_location` — blank-name guard returns `None` without any write; otherwise updates `name` and returns the refreshed row.
- `delete_workspace_location` — counts bound channels first (for the audit event's `cascaded_channel_count`), then issues a `DELETE` and relies on the migration-007 FK cascade to clean up `channel_locations`. Returns `True` iff a row was deleted.
- `set_default_workspace_location` — existence-checks the target id **before** clearing the existing default so an invalid id can never leave the workspace with zero defaults. Inside a single pooled connection/transaction it clears all `is_default=TRUE` rows then sets `is_default=TRUE` on the target.

A new test module `tests/test_workspace_client_offices.py` exercises all three helpers, including the FK cascade on delete and the "missing id leaves previous default intact" invariant for set_default.

## Decisions Made

- **D-08 (resolver contract):** Hard-removed the auto-bind fallback rather than gating it behind a feature flag. Simpler, and plan 05 will land the always-prompt UI immediately after this plan.
- **D-17 (exactly one default):** Existence-check BEFORE clearing `is_default=FALSE`, not after. Prevents an invalid `location_id` from wiping the current default silently.
- **D-23 (cascade):** Relied on the migration-007 FK `ON DELETE CASCADE` instead of issuing a manual `DELETE FROM channel_locations`. Simpler and guaranteed atomic.
- **D-24 (audit):** `structlog.get_logger(__name__)` instance named `_office_log` so the existing stdlib `logger` stays for the legacy log lines.
