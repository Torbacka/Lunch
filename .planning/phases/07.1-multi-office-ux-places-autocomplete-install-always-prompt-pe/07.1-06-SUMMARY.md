# Phase 07.1 Plan 06: App Home Offices Section Summary

Replaced the single Location row in App Home State B with a full Offices management section: list rendering from `list_workspace_locations`, per-row Rename / Set-default / Delete buttons gated on Slack workspace admin, always-available "Add office" button that reuses the plan-05 `build_add_office_modal`, and wired admin-gated view_submission handlers for rename/delete. Plan 02's stubs (`app_home_service.py:213` reading `settings.get('location')` and `ACTION_EDIT_LOCATION` modal receiving `current_location=None`) are now fully resolved.

## What Shipped

### Task 1 — Offices section + rename/delete modal builders

**`lunchbot/services/app_home_service.py`**
- Removed legacy surface: `ACTION_EDIT_LOCATION`, `CALLBACK_LOCATION`, `build_location_modal`, and the `_build_state_b` "Location row" block.
- New constants: `ACTION_ADD_OFFICE_FROM_HOME`, `ACTION_RENAME_OFFICE`, `ACTION_SET_DEFAULT_OFFICE`, `ACTION_DELETE_OFFICE`, `CALLBACK_RENAME_OFFICE`, `CALLBACK_DELETE_OFFICE`.
- `build_home_view(settings, is_admin=True, locations=None)` — new keyword-only `locations` argument, defaults to `[]`. Module remains DB-free (caller fetches).
- `_build_offices_section(locations, is_admin)` — renders header, always-visible Add button, per-row section with `:star:` default badge + `lat_lng` secondary line, and (admin-only) per-row actions block with Rename / Set default / Delete. "Set default" hidden on the already-default row. Empty-state: context line `_No offices yet — add one to get started._` + Add button.
- `build_rename_office_modal(team_id, location_id, current_name)` — plain_text_input block, private_metadata `{team_id, location_id}`, prefilled name.
- `build_delete_office_modal(team_id, location_id, current_name)` — danger-styled confirmation, private_metadata `{team_id, location_id, name}`, warning copy explains that bound channels will be unbound.

**`tests/test_app_home_offices.py` (new, 11 tests)** — rendering admin/non-admin/empty, legacy symbol absence, rename/delete modal shapes, block_actions admin gate (both branches), view_submission rename + delete.

### Task 2 — Admin gate + action/view_submission wiring

**`lunchbot/client/slack_client.py`**
- Added `is_workspace_admin(user_id, team_id)` — calls `users.info` with the workspace bot token and returns `is_admin or is_owner`. Returns `False` on any exception (least-privilege, T-07.1-26).

**`lunchbot/blueprints/slack_actions.py`**
- Imports: dropped `build_location_modal` / `ACTION_EDIT_LOCATION` / `CALLBACK_LOCATION`; added the six new constants + `build_rename_office_modal` / `build_delete_office_modal` + `rename_workspace_location` / `delete_workspace_location` / `set_default_workspace_location`.
- `_APP_HOME_ACTIONS` frozenset updated (ACTION_EDIT_LOCATION out, four new office actions in).
- `_handle_block_actions` now forwards `user_id` and `first_action` to `_handle_app_home_action` so the admin-gated branches can enforce Slack admin status and read the clicked location id.
- `_handle_app_home_action` dropped the ACTION_EDIT_LOCATION branch and added:
  - ACTION_ADD_OFFICE_FROM_HOME — no admin gate (D-07/D-15), opens `build_add_office_modal(team_id, channel_id=None)`.
  - ACTION_RENAME_OFFICE / ACTION_SET_DEFAULT_OFFICE / ACTION_DELETE_OFFICE — `is_workspace_admin` gate; non-admins log `app_home_admin_gate_blocked` and return 200 silently. Target row existence-checked via `list_workspace_locations` before opening modal or mutating. Set default calls `set_default_workspace_location` and immediately refreshes App Home.
- `_handle_view_submission`: dropped CALLBACK_LOCATION branch. Added CALLBACK_RENAME_OFFICE (admin re-check + empty-name guard → `rename_workspace_location` + refresh) and CALLBACK_DELETE_OFFICE (admin re-check → `delete_workspace_location` + refresh). Defense in depth (T-07.1-25): modal submissions re-check admin status even though non-admins never see the buttons.
- CALLBACK_ADD_OFFICE (plan 05) extended: when `channel_id` is absent (App Home context), `_refresh_app_home(user_id, team_id)` is called after creating the office so the new row appears immediately.
- `_refresh_app_home` now fetches locations via `list_workspace_locations` and derives `is_admin` from `slack_client.is_workspace_admin` instead of hard-coding `True`.

**`lunchbot/blueprints/events.py`**
- `app_home_opened` handler now uses `slack_client.is_workspace_admin` (the blueprint-local `_is_workspace_admin` helper remains in the file but is no longer called — it predates the central helper; leaving it keeps the diff minimal).
- Imports `list_workspace_locations`; passes `locations=list_workspace_locations(team_id) or []` to `build_home_view`.

**`tests/test_app_home.py` (updated)** — three legacy tests in `TestBuildHomeViewStateB` adjusted to reference the new Offices surface:
- `test_state_b_shows_edit_buttons` — asserts `ACTION_ADD_OFFICE_FROM_HOME` in blocks instead of `ACTION_EDIT_LOCATION` (which no longer exists).
- `test_state_b_non_admin_no_buttons` — refined: explicitly checks the admin-only actions are absent AND the single "Add office" actions block IS present (the Add button is always shown, D-07/D-15).
- `test_state_b_location_display` / `test_state_b_no_location` — replaced with `test_state_b_offices_section_rendered` / `test_state_b_offices_section_empty_state`.
- `TestBuildLocationModal` class removed (legacy modal gone).

## Threat Mitigations Applied

| Threat | Mitigation |
|---|---|
| T-07.1-25 Elevation of Privilege (rename/delete/set_default) | Two-layer defense: (a) buttons not rendered for non-admins (rendering tests assert), (b) `_handle_app_home_action` re-checks `is_workspace_admin` before opening modals / mutating, (c) `_handle_view_submission` re-checks on rename + delete submissions. `grep -c is_workspace_admin lunchbot/blueprints/slack_actions.py` → 4. |
| T-07.1-26 Info disclosure on admin-check failure | `is_workspace_admin` catches all exceptions and returns `False` — least-privilege default. |
| T-07.1-27 Tampering with location_id in private_metadata | All handlers `int()`-cast before passing to workspace_client helpers; helpers run under `_set_tenant` + `team_id` filter so cross-tenant forgery is blocked at the DB via RLS. |
| T-07.1-28 Spoofed Set-default click by non-admin | Handler-level gate catches this even if a malicious client crafts a block_actions payload. |
| T-07.1-29 DoS via repeated users.info on every refresh | Accepted — Slack Tier 4 rate limits are generous, App Home publishes are infrequent. Caching deferred. |
| T-07.1-30 Repudiation of admin mutations | workspace_client helpers (plan 02) already emit `office_rename` / `office_delete` / `office_set_default` structlog events with `team_id, location_id, actor_user_id`; slack_actions does NOT log again. |

## Decisions Realized

- **D-12 (offices list):** `_build_offices_section` renders `list_workspace_locations(team_id)` output 1:1.
- **D-13 (per-row buttons):** Rename / Set default / Delete on every row, with Set default suppressed on the already-default row.
- **D-14 (reuse Add-office modal):** Handler calls `office_modal_service.build_add_office_modal(team_id, channel_id=None)` — the same builder plan 05 uses for the `/lunch` prompt.
- **D-15 (admin gate on mutations, not on Add):** Rename / Delete / Set default check `is_workspace_admin`; Add office does not.
- **D-16 (delete cascade):** Relies on `delete_workspace_location` (plan 02) which uses the migration-007 FK `ON DELETE CASCADE` — no manual unbind in slack_actions.
- **D-17 (exactly one default):** Relies on `set_default_workspace_location` (plan 02) which existence-checks before clearing the old default.
- **D-18 (rename name-only):** Relies on `rename_workspace_location` (plan 02) with its blank-name guard returning `None`.
- **D-24 (audit log):** All mutations emit `office_<op>` structlog events from the plan-02 helpers. slack_actions adds only the `app_home_admin_gate_blocked` warning for defense-in-depth observability.
- **D-25 (events.py path covered):** `events.py` now calls `list_workspace_locations` and the shared admin helper. Tests cover both block_actions and view_submission branches.
